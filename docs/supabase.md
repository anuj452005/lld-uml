# Supabase

Supabase is the open-source Firebase alternative, providing a complete backend development platform built on PostgreSQL. It offers a hosted Postgres database with real-time subscriptions, authentication and authorization, auto-generated REST and GraphQL APIs, file storage, edge functions, and vector/AI capabilities. The platform combines enterprise-grade open-source tools to deliver a Firebase-like developer experience while maintaining full database access and control.

The platform supports multiple client libraries including JavaScript/TypeScript, Flutter/Dart, Swift, Kotlin, and Python, with both official and community-maintained SDKs. Supabase can be used as a hosted cloud service or self-hosted, making it suitable for projects ranging from rapid prototypes to production applications requiring full data sovereignty.

## Installation and Setup

Install the Supabase JavaScript client library to interact with your Supabase project from any JavaScript or TypeScript application.

```bash
# Install via npm
npm install @supabase/supabase-js

# Install via yarn
yarn add @supabase/supabase-js

# Install via pnpm
pnpm add @supabase/supabase-js
```

```javascript
// Initialize the Supabase client
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = 'https://your-project.supabase.co'
const supabaseKey = 'your-anon-key'
const supabase = createClient(supabaseUrl, supabaseKey)
```

## Database: Creating Tables

Create database tables using SQL with proper data types, primary keys, and foreign key relationships to build your application's data model.

```sql
-- Create a movies table with auto-generated primary key
create table movies (
  id bigint generated by default as identity primary key,
  name text,
  description text
);

-- Create related tables with foreign keys
create table categories (
  id bigint generated always as identity primary key,
  name text
);

alter table movies
  add column category_id bigint references categories;

-- Create many-to-many relationships with join tables
create table actors (
  id bigint generated by default as identity primary key,
  name text
);

create table performances (
  id bigint generated by default as identity primary key,
  movie_id bigint not null references movies,
  actor_id bigint not null references actors
);
```

## Database: Querying Data

Query data from tables using the fluent API with support for filtering, selecting specific columns, ordering, and pagination.

```javascript
// Basic select query
const { data: instruments } = await supabase.from('instruments').select()

// Select with filters and ordering
const { data: movies, error } = await supabase
  .from('movies')
  .select('id, name, description')
  .eq('category_id', 1)
  .order('name', { ascending: true })
  .limit(10)

// Select with foreign key relationships (joins)
const { data: moviesWithCategories } = await supabase
  .from('movies')
  .select(`
    id,
    name,
    categories (
      name
    )
  `)

// Full-text search
const { data: searchResults } = await supabase
  .from('movies')
  .select()
  .textSearch('description', 'Jedi training')
```

## Database: Views

Create SQL views to simplify complex queries and provide a convenient interface for frequently accessed data combinations.

```sql
-- Create a view joining multiple tables
create view transcripts as
  select
    students.name,
    students.type,
    courses.title,
    courses.code,
    grades.result
  from grades
  left join students on grades.student_id = students.id
  left join courses on grades.course_id = courses.id;

-- Grant access to the view
grant all on table transcripts to authenticated;

-- Query the view
select * from transcripts;

-- Create a materialized view for better performance
create materialized view transcripts_cached as
  select
    students.name,
    students.type,
    courses.title,
    courses.code,
    grades.result
  from grades
  left join students on grades.student_id = students.id
  left join courses on grades.course_id = courses.id;

-- Refresh materialized view when needed
refresh materialized view transcripts_cached;
```

## Database: Database Functions

Create PostgreSQL functions that can be called from the client library using RPC for complex operations and business logic.

```sql
-- Simple function returning text
create or replace function hello_world()
returns text
language sql
as $$
  select 'hello world';
$$;

-- Function returning a table set
create or replace function get_planets()
returns setof planets
language sql
as $$
  select * from planets;
$$;

-- Function with parameters using plpgsql
create or replace function add_planet(name text)
returns bigint
language plpgsql
as $$
declare
  new_row bigint;
begin
  insert into planets(name)
  values (add_planet.name)
  returning id into new_row;
  return new_row;
end;
$$;
```

```javascript
// Call functions using rpc()
const { data: greeting } = await supabase.rpc('hello_world')
console.log(greeting) // 'hello world'

// Call function with parameters
const { data: newPlanetId } = await supabase.rpc('add_planet', { name: 'Jakku' })

// Call function that returns a table set with filters
const { data: planets } = await supabase.rpc('get_planets').eq('id', 1)
```

## Authentication: Email and Password Sign Up

Register users with email and password authentication with optional email confirmation and redirect URLs.

```javascript
// Sign up a new user
async function signUpNewUser() {
  const { data, error } = await supabase.auth.signUp({
    email: 'user@example.com',
    password: 'secure-password-123',
    options: {
      emailRedirectTo: 'https://example.com/welcome',
    },
  })

  if (error) {
    console.error('Sign up error:', error.message)
    return null
  }

  console.log('User created:', data.user)
  return data.user
}
```

## Row Level Security (RLS)

Enable and configure Row Level Security policies to control data access at the database level based on user authentication.

```sql
-- Enable RLS on a table
alter table profiles enable row level security;

-- Policy: Public read access
create policy "Public profiles are visible to everyone"
on profiles for select
to anon, authenticated
using (true);

-- Policy: Users can only view their own data
create policy "Users can view their own profile"
on profiles for select
using ((select auth.uid()) = user_id);

-- Policy: Users can insert their own data
create policy "Users can create their own profile"
on profiles for insert
with check ((select auth.uid()) = user_id);

-- Policy: Users can update their own data
create policy "Users can update their own profile"
on profiles for update
using ((select auth.uid()) = user_id)
with check ((select auth.uid()) = user_id);

-- Policy: Users can delete their own data
create policy "Users can delete their own profile"
on profiles for delete
using ((select auth.uid()) = user_id);
```

## Realtime: Listening to Database Changes

Subscribe to real-time database changes using Postgres CDC (Change Data Capture) with support for filtering by table, event type, and row values.

```javascript
// Listen to all changes in public schema
const channel = supabase
  .channel('schema-db-changes')
  .on(
    'postgres_changes',
    {
      event: '*',
      schema: 'public',
    },
    (payload) => console.log('Change received:', payload)
  )
  .subscribe()

// Listen to INSERT events on specific table
const insertChannel = supabase
  .channel('table-inserts')
  .on(
    'postgres_changes',
    {
      event: 'INSERT',
      schema: 'public',
      table: 'todos',
    },
    (payload) => console.log('New todo:', payload.new)
  )
  .subscribe()

// Listen with filter
const filteredChannel = supabase
  .channel('filtered-changes')
  .on(
    'postgres_changes',
    {
      event: 'UPDATE',
      schema: 'public',
      table: 'messages',
      filter: 'room_id=eq.123',
    },
    (payload) => console.log('Message updated:', payload)
  )
  .subscribe()

// Cleanup: unsubscribe when done
await supabase.removeChannel(channel)
```

## Realtime: Multiple Subscriptions

Subscribe to multiple tables and event types within a single channel for efficient real-time updates.

```javascript
const channel = supabase
  .channel('db-changes')
  .on(
    'postgres_changes',
    {
      event: '*',
      schema: 'public',
      table: 'messages',
    },
    (payload) => console.log('Message change:', payload)
  )
  .on(
    'postgres_changes',
    {
      event: 'INSERT',
      schema: 'public',
      table: 'users',
    },
    (payload) => console.log('New user:', payload)
  )
  .subscribe((status) => {
    if (status === 'SUBSCRIBED') {
      console.log('Connected to realtime!')
    }
  })
```

```sql
-- Create bucket using SQL
insert into storage.buckets (id, name, public)
values ('avatars', 'avatars', false);

-- Create storage policy for authenticated users
create policy "Users can upload their own avatar"
on storage.objects for insert
to authenticated
with check (bucket_id = 'avatars' and (storage.foldername(name))[1] = auth.uid()::text);

on storage.objects for select
to public
using (bucket_id = 'avatars');
```

## Edge Functions: Creating and Deploying

Create serverless Edge Functions using Deno that run globally on Supabase's edge network for low-latency operations.

```typescript
// supabase/functions/hello-world/index.ts
Deno.serve(async (req) => {
  const { name } = await req.json()
  const data = {
    message: `Hello ${name}!`,
  }

  return new Response(JSON.stringify(data), {
    headers: { 'Content-Type': 'application/json' }
  })
})
```

```bash
# Create a new edge function
supabase functions new hello-world

# Test locally
supabase start
supabase functions serve hello-world

# Deploy to production
supabase functions deploy hello-world

# Test with curl
curl -i --request POST 'http://localhost:54321/functions/v1/hello-world' \
  --header 'Authorization: Bearer YOUR_ANON_KEY' \
  --header 'Content-Type: application/json' \
  --data '{"name":"Functions"}'
```

## Complete React Application Example

Build a complete React application with Supabase for data fetching, authentication, and real-time updates.

```javascript
import { useEffect, useState } from 'react'
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
)

function App() {
  const [user, setUser] = useState(null)
  const [todos, setTodos] = useState([])
  const [newTodo, setNewTodo] = useState('')

  useEffect(() => {
    // Check current auth state
    supabase.auth.getSession().then(({ data: { session } }) => {
      setUser(session?.user ?? null)
    })

    // Listen for auth changes
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (_event, session) => setUser(session?.user ?? null)
    )

    return () => subscription.unsubscribe()
  }, [])

  useEffect(() => {
    if (!user) return

    // Fetch todos
    fetchTodos()

    // Subscribe to realtime changes
    const channel = supabase
      .channel('todos-changes')
      .on(
        'postgres_changes',
        { event: '*', schema: 'public', table: 'todos' },
        () => fetchTodos()
      )
      .subscribe()

    return () => supabase.removeChannel(channel)
  }, [user])

  async function fetchTodos() {
    const { data } = await supabase
      .from('todos')
      .select('*')
      .order('created_at', { ascending: false })
    setTodos(data || [])
  }

  async function addTodo(e) {
    e.preventDefault()
    if (!newTodo.trim()) return

    await supabase.from('todos').insert({ task: newTodo, user_id: user.id })
    setNewTodo('')
  }

  async function toggleComplete(id, completed) {
    await supabase.from('todos').update({ completed: !completed }).eq('id', id)
  }

  if (!user) {
    return (
      <button onClick={() => supabase.auth.signInWithOAuth({ provider: 'github' })}>
        Sign in with GitHub
      </button>
    )
  }

  return (
    <div>
      <h1>Todo List</h1>
      <form onSubmit={addTodo}>
        <input
          value={newTodo}
          onChange={(e) => setNewTodo(e.target.value)}
          placeholder="Add a new todo"
        />
        <button type="submit">Add</button>
      </form>
      <ul>
        {todos.map((todo) => (
          <li key={todo.id}>
            <input
              type="checkbox"
              checked={todo.completed}
              onChange={() => toggleComplete(todo.id, todo.completed)}
            />
            <span style={{ textDecoration: todo.completed ? 'line-through' : 'none' }}>
              {todo.task}
            </span>
          </li>
        ))}
      </ul>
      <button onClick={() => supabase.auth.signOut()}>Sign out</button>
    </div>
  )
}

export default App
```

## Summary

Supabase provides a comprehensive backend platform that combines PostgreSQL's power with modern developer tooling. The main use cases include building full-stack applications with real-time features, implementing secure user authentication with multiple providers, managing file uploads and media storage, and creating serverless API endpoints with Edge Functions. The platform excels at rapid prototyping while maintaining production-grade capabilities through its auto-generated APIs, Row Level Security, and real-time subscriptions.

Integration patterns typically involve initializing the Supabase client with project credentials, using the fluent API for database operations, implementing RLS policies for security, and subscribing to real-time channels for live updates. The modular architecture allows developers to use only the features they need while maintaining consistency across JavaScript, Flutter, Swift, Kotlin, and Python client libraries. Whether building a simple CRUD application or a complex multi-tenant SaaS platform, Supabase provides the infrastructure and developer experience to move quickly without sacrificing security or scalability.
